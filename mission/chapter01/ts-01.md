# 🍠 추가 미션. type vs interface
## 공통점
둘 다 변수, 객체, 함수 등 타입 모양을 정할 때 쓸 수 있음
```
const user = {
  name: "서현",
  age: 23,
};
```
이러한 객체가 있을 때 type, interface 둘 다 표현할 수 있음
### type
```
type User = {
  name: string;
  age: number;
};
```

### interface
```
interface User {
  name: string;
  age: number;
}
```

## 차이점
### 타입 정의의 범용성
- interface는 주로 객체 형태를 정의
    ```
    interface Student {
    name: string;
    grade: number;
    }
    ```
- type은 더 다양한 타입을 정의
    - 유니온 타입
        ```
        type Status = "success" | "error" | "loading";
        ```
    - 튜플
        ```
        type Point = [number, number];
        ```
    - 함수 타입
        ```
        type Add = (a: number, b: number) => number;
        ```
### 선언 병합
- interface는 같은 이름으로 여러 번 선언하면 합쳐짐
```
interface User {
  name: string;
}

interface User {
  age: number;
}
```
이면
```
interface User {
  name: string;
  age: number;
}
```
- type은 선언병합이 불가능
```
type User = {
  name: string;
};

type User = {
  age: number;
};
```

### 확장 방식 표현
- interface 확장  
extends를 사용
```
interface Person {
  name: string;
}

interface Student extends Person {
  grade: number;
}
```
- type 확장  
$를 사용
```
type Person = {
  name: string;
};

type Student = Person & {
  grade: number;
};
```
