- null과 undefined의 차이점에 대해 직접 작성해주세요 🍠

    - null

        null은 무언가를 가리키고(값이 할당되어) 있지만 해당 대상(할당된 값)이 아무것도 가지고 있지 않다고 명시한 것 입니다.
    
    - undefined
        undefined는 null과 다르게 값이 아직 할당 되지 않아 아무것도 가리키고 있지 않는 것을 표현한 것입니다

- 타입 스크립트에만 존재하는 타입 🍠

    - any 🍠
        
        해당 타입을 설정한 객체에는 어떤 타입이던 허용하겠다 라는 의미입니다.
        
    - unknown 🍠
        
        any와 비슷하게 해당 객체에 어떤 타입의 데이터를 할당하여도 허용하지만 해당 타입을 가진 객체가 어떤 타입인지 확인하기 전까지는 사용을 막습니다.
        
    - void 🍠
        
        함수의 리턴값이 없을 때 사용합니다
        
    - never 🍠
        
        무한 루프나 항상 에러를 던지는 함수 같이 값이 절대 발생 할 수 없을 때 사용합니다.

- 함수에서의 TypeScript 🍠

    - 함수 선언식의 특징에 대해 정리해주세요!
        
        함수 선언식은 코드 상에서 어디에 있던 최상단으로 끌어올려져 함수를 선언하기 전에 (코드 상으로 위에서) 호출이 가능하게는 호이스팅 기능을 지원합니다.
        
        함수 인자를 유연하게 다룰 수 있는 선택적 매개변수 및 기본값 기능을 제공합니다.
        
        - **선택적 매개변수 (`?`)**: 인자가 들어오지 않을 경우 `undefined`로 취급합니다. 반드시 필수 매개변수 뒤에 위치해야 합니다.
        - **기본값 (Default)**: 값이 전달되지 않았을 때 사용할 기본값을 지정합니다.
        
        동일한 함수 이름으로 여러 가지 매개변수 조합을 정의할 수 있는 함수 오버로딩 기능을 지원합니다
        
    - 화살표 함수의 특징에 대해 정리해주세요!
        
        함수 본문이 한 줄인 경우 `return` 과 중괄호를 생략할 수 있습니다.
        
        함수 호이스팅 불가
        
        - 함수 선언식과 달리 화살표 함수는 **변수에 할당하는 방식(함수 표현식)**이기 때문에, 선언하기 전에 호출하면 에러가 발생합니다
