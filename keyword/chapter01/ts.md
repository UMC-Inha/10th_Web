# Typescript

### 장점

1. **안정성 보장** – 타입 검사로 런타임 오류를 줄여줌
2. **개발 생산성 향상** – 자동 완성과 코드 인텔리센스가 강력함
3. **협업에 유리** – 명확한 타입 덕분에 팀원 간 코드 이해가 쉬움
4. **점진적 적용 가능** – 기존 JS 코드에 천천히 TS를 도입할 수 있음

### 함수의 매개변수 타입과 반환 타입

```ts
function add(a: number, b: number): number {
  return a + b;
}

function greet(name: string): string {
  return `안녕하세요, ${name}입니다.`;
}
```

### 함수 선언식

- `function` 키워드로 함수를 선언하는 방식
- 함수명이 있어서 호이스팅이 일어나기 때문에 선언 전에 호출해도 동작할 수 있음
- `this` 바인딩이 호출 방식에 의해 결정되는 일반 함수 형태라서 메서드로 쓸 때 동작이 예측하기 쉬움
- `new`로 생성자처럼 사용할 수 있는 경우가 있어 클래스/프로토타입 문맥에서 활용 가능

### 화살표 함수

- `() => {}` 형태로 간단히 쓰는 함수 표현식
- 호이스팅이 없고 보통 선언 위치 이후에만 사용할 수 있음
- `this`를 자체적으로 가지지 않고 바깥(렉시컬 스코프)의 this를 그대로 사용 그래서 이벤트 핸들러에서 자주 씀
- `arguments`를 따로 만들지 않아 바깥 스코프의 값을 참조함
- 간결한 문법으로 콜백 작업에 특히 적합 (`return` 생략 가능 등)

### 리터럴 타입 (Literal Types)

- 특정한 값 그 자체만을 허용하는 타입

### 객체 리터럴 타입 (Object Literal Types)

- 특정 구조와 값을 가진 객체만 허용하도록 제한하는 타입

### 인덱스 시그니처 (Index Signature)

- 기본적으로 객체 리터럴 타입은 정의된 프로퍼티 외에 다른 프로퍼티를 허용하지 않음
- 하지만 때로는 객체에 추가적인 프로퍼티를 유연하게 넣고 싶을 때가 있음
- 이럴 때 인덱스 시그니처를 사용하면, 임의의 키-값 쌍을 허용할 수 있음

### 선택적 프로퍼티 (Optional Property)

- 객체 리터럴 타입의 프로퍼티 중 일부를 선택적(optional)으로 만들 수 있음

### as const

- 값을 변경 불가능하게 만들고, 리터럴 값 그대로 타입으로 고정하고 싶을 때 사용

### 배열 타입의 문제점

- 실제로는 값이 없는데도, 타입 검증에서 에러를 잡아주지 못함
- 튜플로 해결 가능함

### 튜플 타입의 문제점

- `push`, `pop`, `unshift`, `shift` 같은 배열 메서드는 기본적으로 막지 않음
- 런타임에서는 튜플이 깨질 수 있는 문제가 발생할 수 있음
- `readonly` 키워드를 붙여 튜플을 읽기 전용으로 만들면 해결 가능함

### 유니언 타입 (Union Types)

- 둘 이상의 타입을 허용해서, 변수가 여러 타입 중 하나를 가질 수 있게 함

### 타입 스크립트에만 존재하는 타입

- `any`
  - 모든 타입을 허용하는 타입
  - 타입 검사를 완전히 우회함
  - TS의 타입 안정성을 깨기 때문에 가능하면 사용 지양
- `unknown`
  - 모든 값을 받을 수 있지만 바로 사용할 수 없는 타입
  - 사용하려면 타입 좁히기(type narrowing) 필요
  - any보다 안전한 최상위 타입
- `void`
  - 아무 값도 반환하지 않는 함수의 반환 타입
  - 반환값이 없다는 의미 (undefined와는 다름)
- `never`
  - 절대 반환되지 않는 값의 타입
  - 함수가 끝까지 도달할 수 없는 경우
  - e.g., 항상 에러를 던지는 함수, 무한 루프

### 타입 별칭 (Type Aliases)

- 특정 타입이나 인터페이스를 참조할 수 있는 타입 변수
- 긴 타입이나 복잡한 타입을 간단한 이름으로 정리해서 재사용할 수 있음

### Interface

- 인터페이스 병합
  - 같은 이름을 가진 여러 인터페이스가 선언되면 자동으로 병합됨
- 네임스페이스
  - 코드를 모듈화하고 그룹화해서 관리할 수 있는 방법

### 제네릭 (Generic)

- 다양한 타입을 재사용하기 위해 사용하는 문법

### Enum

- 이름이 있는 상수 집합을 정의하는 방법

1. 숫자형 Enum (Numeric Enum)
2. 문자열 Enum (String Enum)

### Utility Type

| 유틸리티 타입    | 설명                                  |
| ---------------- | ------------------------------------- |
| `Partial<T>`     | 모든 프로퍼티를 선택적으로 만듦 (`?`) |
| `Required<T>`    | 모든 프로퍼티를 필수로 만듦 (`!`)     |
| `Readonly<T>`    | 모든 프로퍼티를 읽기 전용으로 만듦    |
| `Record<K, T>`   | 객체의 키와 값 타입을 정의            |
| `Pick<T, K>`     | 특정 프로퍼티만 선택                  |
| `Omit<T, K>`     | 특정 프로퍼티 제외                    |
| `Exclude<T, U>`  | 특정 타입 제거                        |
| `Extract<T, U>`  | 특정 타입만 추출                      |
| `NonNullable<T>` | `null` 및 `undefined` 제거            |
| `ReturnType<T>`  | 함수의 반환 타입 추출                 |
| `Parameters<T>`  | 함수의 매개변수 타입을 튜플로 추출    |
