# HTML 키워드 정리

## [시맨틱 태그란?] 기타 태그 추가 정리해보기 🍠

### **폼 관련 태그들 :**

- `<form>` : 사용자 입력 전체를 감싸는 양식
- `<input>` : 다양한 입력 필드
- `<button>` : 클릭 가능한 버튼
- `<select>`, `<option>` : 드롭다운 선택 메뉴
- `<textarea>` : 여러 줄 입력
- `<label>` : 입력 요소 설명 (접근성 향상)

---

### **텍스트 관련 태그들 :**

### ✔️ 의미 있는 강조

- `<strong>` : 중요한 텍스트 (굵게 + 의미)
- `<em>` : 강조 텍스트 (기울임 + 의미)

### ✔️ 단순 스타일

- `<b>` : 시각적으로 굵은 글자
- `<i>` : 시각적으로 기울인 글자

### ✔️ 기타 텍스트

- `<cite>` : 저작물의 제목
- `<mark>` : 하이라이트된 텍스트
- `<small>` : 부가 정보 (작은 글씨)
- `<sub>` : 아래 첨자
- `<sup>` : 위 첨자
- `<abbr>` : 줄임말(약어)
- `<code>` : 소스 코드
- `<q>` : 짧은 인용문
- `<blockquote>` : 긴 인용문
- `<pre>` : 공백/줄바꿈 유지
- `<address>` : 연락처/작성자 정보

---

### **제목 / 문단 관련 태그들 :**

- `<h1> ~ <h6>` : 제목
- `<p>` : 문단
- `<br>` : 줄바꿈
- `<hr>` : 가로 구분선

---

### **목록 관련 태그들 :**

- `<ol>` : 순서 있는 목록
- `<ul>` : 순서 없는 목록
- `<li>` : 목록 항목
- `<dl>` : 설명 목록
- `<dt>` : 용어
- `<dd>` : 용어 설명

---

### **표 관련 태그들 :**

- `<table>` : 표 전체
- `<caption>` : 표 제목
- `<tr>` : 행
- `<td>` : 데이터 셀
- `<th>` : 제목 셀
- `<thead>` : 표 머리
- `<tbody>` : 표 본문
- `<tfoot>` : 표 요약
- `<col>` : 열 속성 지정
- `<colgroup>` : 열 그룹 정의

---

### **이미지 / 미디어 관련 태그들 :**

- `<img>` : 이미지
- `<audio>` : 오디오
- `<video>` : 영상
- `<figure>` : 미디어 묶음
- `<figcaption>` : 설명
- `<picture>` : 반응형 이미지
- `<object>` : 외부 리소스
- `<embed>` : 외부 콘텐츠

---

### **레이아웃 태그들 :**

- `<div>` : 의미 없는 블록 영역
- `<span>` : 의미 없는 인라인 영역

---

### **시맨틱 구조 태그들 :**

- `<header>` : 상단 영역
- `<nav>` : 메뉴 영역
- `<main>` : 주요 콘텐츠 (문서당 1개)
- `<section>` : 주제별 구역
- `<article>` : 독립 콘텐츠
- `<footer>` : 하단 영역

<br>

## [body 태그 (Semantic Tag 활용)] Semantic Tag를 잘 활용하였을 때 장점은? 🍠

### 1️⃣ 구조를 한눈에 이해하기 쉬움

- `<div>`만 쓰면 역할 파악이 어려움
- `<header>`, `<nav>`, `<main>`와 같은 **시맨틱 태그**를 사용하면 <br>
  👉 태그만 보고 구조를 바로 이해 가능

🔹 가독성 ↑ / 유지보수 쉬워짐

---

### 2️⃣ 협업 효율 상승

- 다른 개발자가 봐도 역할 및 의미를 빠르게 파악 가능

🔹 코드 해석 시간 ↓ / 협업 효율 ↑

---

### 3️⃣ SEO (검색 엔진 최적화) 향상

- 검색 엔진이 HTML 구조를 분석할 때<br>
  👉 본문 / 메뉴 / 부가 정보 구분이 쉬워짐

🔹 검색 엔진 이해도 ↑

---

### 4️⃣ 웹 접근성이 향상

- 스크린 리더 등 보조 기술이 <br>
  👉 태그 의미 기반으로 구조를 해석

🔹 사용자 경험 ↑ (특히 장애 사용자)

---

### 5️⃣ 유지보수가 용이

- `<div class="box1">` 같은 구조는 나중에 헷갈림
- 시맨틱 태그는 태그 자체가 역할 설명

🔹 수정/확장 쉬움

---

### 6️⃣ CSS / JS 적용이 직관적

예를 들어:

```html
<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>
```

👉 구조가 명확해서

🔹 스타일링 & 기능 적용이 쉬움

## [태그 정리 - element vs container] Element Level과 Container Level은 무엇을 의미하며, 어떤 것이 다른지 정리해주세요.

### 1️⃣ Element Level

Element Level은 **개별 요소 자체**에 초점을 두는 개념으로, <br>
화면에 표시되는 **하나의 독립적인 요소**를 의미한다.

예시:

- `<h1>`
- `<p>`
- `<a>`
- `<img>`
- `<button>`
- `<span>`

이런 태그들은 각각 자기만의 의미를 가진다.

- `<p>` : 문단
- `<a>` : 링크
- `<img>` : 이미지
- `<button>` : 버튼

👉 즉, **하나의 태그가 하나의 역할을 수행하는 것**이 핵심

---

### 2️⃣ Container Level

Container Level은 **다른 요소들을 담고 묶는 영역**에 초점을 두는 개념으로, <br>
여러 요소를 감싸서 **구조를 만들고 그룹화하는 역할**을 한다.

예시:

- `<div>`
- `<section>`
- `<article>`
- `<main>`
- `<header>`
- `<footer>`
- `<nav>`
- `<aside>`

이런 태그들은 보통 자기 자신이 직접 내용을 표현하기보다는,

👉 **안에 들어가는 여러 요소를 묶어주는 틀** 역할을 한다.

---

### 🔍 Element Level과 Container Level의 차이

| 구분 | 🧩 Element Level                   | 🧱 Container Level           |
| ---- | ---------------------------------- | ---------------------------- |
| 개념 | 개별 콘텐츠 자체가 중요            | 여러 요소를 묶는 구조가 중요 |
| 역할 | 하나의 의미 있는 요소 표현         | 문서의 구역과 레이아웃 담당  |
| 특징 | 텍스트, 이미지, 버튼 같은 **부품** | 여러 요소를 담는 **틀**      |
| 중심 | 🔹 내용 중심                       | 🔹 구조 중심                 |

---

### 📄 예시 코드

```html
<header>
  <h1>카사 블로그</h1>
  <nav>
    <a href="#">홈</a>
    <a href="#">소개</a>
  </nav>
</header>

<main>
  <section>
    <h2>20260316</h2>
    <p>오늘은 떡볶이를 먹었당</p>
    <button>+1kg</button>
  </section>
</main>
```

위 코드에서의 구분

| 🧱 Container Level                         | 🧩 Element Level                         |
| ------------------------------------------ | ---------------------------------------- |
| `<header>`, `<nav>`, `<main>`, `<section>` | `<h1>`, `<h2>`, `<a>`, `<p>`, `<button>` |

<br>

## [태그 정리 - block vs inline] inline-block은 어떠한 방식으로 동작할까요? 🍠

### inline-block

inline-block은 inline처럼 옆으로 배치되지만, <br>
**block처럼 크기 지정이 가능한 방식**이다.

- 옆으로 나란히 놓고 싶고
- width, height도 주고 싶을 때

  👉 자주 사용한다.

🔹 특징

- 줄바꿈 없이 옆으로 배치
- 내용 크기만큼 차지하지만, width, height 지정 가능
- margin, padding 상하좌우 적용 가능

---

### 📄 예시 코드

```html
<button>버튼1</button> <button>버튼2</button>
```

👉 버튼은 대표적인 inline-block 성격을 가지는 요소이며 <br>
👉 옆으로 배치되면서도 크기 조절이 가능하다.

또는

```css
span {
  display: inline-block;
  width: 100px;
  height: 50px;
}
```

👉 span도 inline처럼 배치되지만 크기를 가질 수 있다.

---

### 🔍 block vs inline vs inline-block 비교

| 구분         | 줄바꿈 | 한 줄 전체 차지 | width / height 지정 | 옆으로 배치 |
| ------------ | ------ | --------------- | ------------------- | ----------- |
| block        | O      | O               | O                   | X           |
| inline       | X      | X               | X                   | O           |
| inline-block | X      | X               | O                   | O           |
