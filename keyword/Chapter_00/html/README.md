## HTML의 다양한 태그들

**텍스트**

- `<h1>` : 가장 큰 제목, 페이지에서 제일 중요한 제목에 사용
- `<h2>` : 중간 제목, `<h1>`보다 한 단계 아래 제목
- `<h3>` : 세부 제목, `<h2>` 보다 한 단계 아래 제목 (`<h6>` 까지 존재)
- `<p>` : 문단 (paragraphy), 일반적인 본문 글을 쓸 때 사용
- `<span>` : 큰 의미는 없고 글자 일부만 묶고 싶을 때 사용
    - 줄 바꿈이 안 되기 때문에 텍스트 일부만 꾸밀 때 주로 사용
- `<br />` : 줄바꿈
- `<label>` : 입력창이 무엇을 위한 건지 설명해주는 태그
    - 주로 `input`, `textarea`, `select`와 같은 폼 태그 앞에 함께 씀
    - 예시
        
        ```jsx
        <label for="email">이메일</label>
        <input id="email" type="email" />
        ```
        
- `<script>` : js 코드를 넣거나 불러오는 태그
- `<iframe>` : 다른 웹페이지를 현재 페이지 안에 끼워 넣는 태그
    
    ```jsx
    <iframe src="https://example.com"></iframe>
    ```
    
- `<strong>` : 굵게 보여짐 (중요한 내용을 담은 경우)
- `<b>` : 단순히 굵게 보여짐
- `<em>` : 기울여져서 보여짐 (강조해야 하는 내용을 담은 경우)
- `<i>` : 단순히 기울여져서 보여짐
    
    <aside>
    💡
    
    중요한 내용 / 강조해야 하는 내용은 누구에게 알려질까?
    
    - 브라우저 / HTML 해석기
    - 스크린리더
    - 검색엔진 (SEO)
    </aside>
    

**목록**

- `<ul>` : 순서 없는 목록 (앞에 - 가 붙음)
- `<ol>` : 순서 있는 목록 (앞에 번호가 붙음)
- `<li>` : `<ul>`, `<ol>` 안에서 쓰는 목록 각각의 항목

**표**

- `<table>` : 표 전체를 감싸는 태그
- `<thead>` : 표의 머리 부분 (제목)
- `<tbody>` : 표의 본문 부분 (실제 데이터)
- `<tfoot>` : 표의 마무리 부분 (합계, 요약 등)
- `<th>` : 표의 제목, 굵게 표시
- `<tr>` : 표의 한 줄
- `<td>` : 표의 일반적인 데이터 칸

```html
// 표 관련 태그들의 예시
<table>
	<thead>
		<tr>
			<th>이름</th>
			<th>나이</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>서현</td>
			<td>25</td>
		</tr>
		<tr>
			<td>엘리</td>
			<td>25</td>
		</tr>
	</tbody>
	<tfoot>
		<tr>
			<td>총</td>
			<td>2명</td>
		</tr>
	</tfoot>
</table>
```

**입력**

- `<textarea>` : 여러 줄 입력 칸
    - `<input>`은 한 줄만 입력되기에 긴 글을 적을 때 사용

## 시멘틱 태그를 잘 활용했을 때의 장점
- 개발자가 개발할 때 태그 이름만으로 내용/위치를 이해하기 쉬워짐
    
    코드 가독성이 좋아지고, 유지보수가 쉬워짐
    
- 검색엔진 최적화(SEO)에 유리
    
    검색엔진은 HTML 구조를 보고 페이지 내용을 분석하는데, 시멘틱 태그를 쓰면 검색엔진이 이 문서의 중요한 부분이 어디인지 더 잘 파악할 수 있음
    
    좋은 구조를 만들면 검색엔진에 유리해져 검색 순위가 오를 수 있음
    
- 웹 접근성에 도움
    
    스크린리더 같은 보조기기가 페이지를 읽을 때 시멘틱 태그를 기준으로 읽기 때문에 시각장애인 같은 보조기기를 이용하는 사용자도 페이지를 이해하기 쉬워짐



## Element Level과 Container Level
### Element Level
태그 개별 요소 자체
안에 들어가는 내용물이 중요함
### Container Level
여러 요소를 감싸고 묶고 배치하는 역할
안에 들어가는 내용 하나하나보다, 그 내용을 어떤 구역에 넣고 어떻게 정리할지를 보는 것이 핵심
- 예시
```html
<section>
	<h1>공지사항</h1>
	<p>이번 주 일정입니다.</p>
</section>
```

여기서 `<h1>`, `<p>` 는 개별 요소(element level)이고, `<section>`은 개별 요소 둘을 감싸고 하나의 의미있는 구역으로 묶는 역할(container level)
