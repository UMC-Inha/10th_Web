## JSX는 반드시 하나의 태그만 반환해야 한다

    React 컴포넌트에서 JSX를 반환할 때는 **무조건 하나의 부모 태그**로 감싸야 해요.
    그러면, 위와 같이 **여러 개의 태그를 동시에 반환하려고 할 때**는 어떻게 해야 할까요?

    - 답변 🍠
        
        ```tsx
        // 코드 아래 첨부
        function App() {
        return (
            <React.Fragment> 
                <strong>상명대학교</strong>
                <p>매튜/김용민</p>
            </React.Fragment> 
        )
        };
        export default App
        ```
        
    <aside>
    🍠

    이유: 

    JSX는 반드시 하나의 태그만을 반환해야 하기에
    여러개의 태그를 동시에 반환하려고 할 때는 하나의 부모 태그로 감싸야 한다.
    하지만 <div>같은 태그를 사용하면 굳이 불필요한 DOM요소가 늘어나니
    <>라는 빈 태그로 묶어 레이아웃에 영향 없이 깔끔하게 렌더링 가능합니다

</aside>

## 구조분해 할당 활용 
    - 답변 🍠
        ```ts
            interface prop{
                tech:string
            }

            export const List = ({tech}:prop) => {
                return <li>{tech}</li>
                
            };
        ```
    함수 내부에서 구조 분해 할당을 적용하여 prop.tech 방식으로 접근하지 않고 바로 tech라는 이름으로 접근 할 수가 있습니다.

## Lazy Initialization(게으른 초기화)  
    useState의 인자로 값을 넣어 초기화 할 경우 매 렌더링 마다 연산이 일어나기에 이를 해결하기 위해
    인자로 값 대신 함수를 전달하여 최초 렌더링 시에만 값을 할당하고 재 랜더링 시에는 무시하는 방법

    첫 렌더링 이후에는 연산을 하지 않아 성능적인 이점이 있으나 최초 렌더링 시에 지연이 발생할 수 있다 → ux에 영향이 있을 수 있다
    코드 복잡도가 증가하여 유지보수에 불리할 수 있다

    사용처:연동시 대량의 데이터를 로드할 경우 등 대량의 정보를 관리해야할때 사용하기 좋다
